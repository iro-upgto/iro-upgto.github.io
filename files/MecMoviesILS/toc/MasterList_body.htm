Using MecMovies on your own Web Page

The MecMovies software is available on the Internet at www.umr.edu/~mecmovie. However, there are many good reasons why it may be preferable for you, the instructor, to install MecMovies on your institution's own Internet site.

To set up MecMovies on your website, copy the contents of this CD to a folder (which is publicly accessible) on your system, maintaining the file folder names and structure as given on the CD. Say, for example, that you create a folder named MM in a publicly accessible portion of your Internet site. Into this MM folder, copy the CD contents. In the root folder (i.e., MM), you'll find a file called index.html. This web page loads and plays the MecMovies content.

Should you wish to install this software on your own website, there are a couple of ways in which MecMovies can be customized to better suit your teaching needs and desires.

Customizing the Gold Menu Button Sequence

You can customize MecMovies to suit your sequence of course topics. In the toc folder, you will find the file:

  • MecMoviesMenuSetup.txt

This file controls the sequence of the gold menu buttons appearing along the left side of MecMovies. You can edit this text file to fit the sequence of your course.

Typical MecMoviesMenuSetup.txt File

&goldmenu1=Home<home.swf>&
&submenu1=<About><Navigation>&

&goldmenu3=Axial<axial.swf>&
&submenu3=<Determinate>
<Axial+w/temperature>
<Indeterminate>
<Indet+w/temperature>
<Indet+w/misfit>&

&goldmenu7=Stress+Transform<stress.swf>&
&submenu7=<Stress+Transform+Eqns>
<Mohr's+Circle+Stress>&

&goldmenu13=Assignments<assignments.swf>&
&submenu13=<UMR Summer 2004>
<UMR Spring 2004>
<UMR Fall 2004>
<Checkpoints Try One>
<Section E>
<Demo One>
<Demo Two>&

&EndOfFile=1

The gold buttons are generated in sequence according to the variables goldmenu1, goldmenu2,...To alter the sequence of buttons, simply renumber the goldmenu variables and the corresponding submenu variables. Referring to the example code above, the Stress Transform button could be placed 3rd in the sequence by changing the variables goldmenu7 and submenu7 to goldmenu3 and submenu3, respectively, and renumbering the subsequent variables.

Syntax of goldmenu variables:

&goldmenu7=Stress+Transform<stress.swf>&

Start and end this line with ampersand symbols. The variable name is goldmenuXX where XX is the position of each gold button (i.e., menu button 1 is at the top). Between the equals sign and the < symbol is the text label that will be placed on the menu button. Customize this to anything you'd like (note: spaces between words can be denoted with a + symbol). Between the < and > symbols is the name of the .swf file that is the table of contents movie associated with the topic (these are .swf files found in the toc folder). Do not change this portion of the goldmenu variable.

Syntax of submenu variables:

The submenu buttons correspond to the table of contents movies. The submenu button labels are found between the < and > symbols. These labels can be changed to anything you'd prefer.

The final line of the MecMoviesMenuSetup.txt file should be this:

&EndOfFile=1&

Creating a Custom Assignment Movie

In the toc folder, there are several text files called:

  • myHomework0.txt
  • myHomework1.txt
  • myHomework2.txt
  • myHomework3.txt
  • myHomework4.txt
  • myHomework5.txt
  • myHomework6.txt

Each file corresponds to one of the submenu buttons under the Assignments menu item. These files control the generation of a customized assignment page in MecMovies. Although you'll probably need only one, up to seven different assignment pages can be generated. For example, individual assignment schedules could be defined for multiple sections of the Mechanics of Materials course at your institution. These are ordinary text files that can be generated by many types of text editors, such as NotePad. If you edit these files in Microsoft Word, make sure to save the file as text (*.txt) rather than as a Word document (*.doc).

The contents of the homework assignment text files is illustrated by the example file reproduced below. In general, the file is formatted as variable_name=value, with each line in the myHomework*.txt file beginning and ending with an ampersand (&) symbol.

The first line should be the heading that will appear at the bottom of the assignment page. The variable name is subtitle. Following the variable name and the equals sign, enter the desired label that will appear on the assignment page. In the example shown, the subtitle BE 110 - UMR Spring 2004 will appear at the bottom of the dynamically-generated assignment page corresponding to this myHomework*.txt file.

A homework assignment page is dynamically-generated according to the instructions coded in the myHomework*.txt file. The page is divided into a grid of 14 rows and 5 columns. A naming scheme identical to the common spreadsheet naming scheme is used to specify grid locations: columns are referred to using letters, rows are referred to with numbers. To define an assignment, you need specify only (1) the button to show, (2) a label for the button, and (3) a code number for the movie to be associated with the button. To illustrate the syntax, consider this line from the example file below:

&C2=<Feb 02c>M440&comment=Gear trains: torque/stress&

The variable name is C2. This tells MecMovies to generate a button in the third column (C) and the second row (2) of the screen grid.. After the equals sign, there is text enclosed by < > symbols. The text inside the < > symbols will be used as the button label. Following the > symbol, there is a four-letter code that identifies a specific movie in the MecMovies collection. In this example, the code is M440. Following the four-letter code, there is an ampersand symbol, which completes the button definition.

To find the four-letter code associated with each movie, use the links in the left frame to browse through the various topics.

For this example, comments have been added to make the text file more readable. These comments are optional; however, they should be preceeded and followed by ampersand symbols, as shown in the example.

To make the dynamically-generated assignment page more organized for your students, you may also define special buttons that are simply labels. These can be positioned anywhere on the grid. To make a label, use LABEL instead of the four-letter code number.

&A1=<CHECKPOINTS>LABEL&comment=Label&

With 14 rows and 5 columns, up to 70 buttons can be dynamically generated for your assignment page. Use any number of buttons you desire, up to 70 max. The buttons can be arranged in any pattern you choose, according to the button addresses (i.e., C2, D7, B14) you specify.

The final line of the myHomework*.txt file should be this:

&EndOfFile=1&
